积木快速开发平台

一直想做一款后台管理系统，看了很多优秀的开源项目但是发现还是感觉开发不够理想。
管理平台有管理和运营需求，需要快速的迭代出需要的功能模块，所以要求短平快。
目前主流快速开发有2种方式：1、模板生成 2、后台根据数据库渲染 ，当然这里边还有前后端分离的项目，个人的感觉是模板生成离实际可用的工作量还是比较多当然好处是灵活性大想怎么改怎么改，甚至还前后端分离，2个人来搞一天。
于是利用空闲休息时间开始自己写一套后台系统。如此有了积木快速开发平台。，她可以用于所有的Web应用程序，如网站管理后台，网站会员中心，CMS，CRM，OA等等，当然，您也可以对她进行深度定制，以做出更强系统。所有前端后台代码封装过后十分精简易上手，出错概率低。
目标有几个：
1、实现基本的表单控件（约11个）基础的视图控件(列表，父子表，树表)
2、相关的功能在此表单控件、视图控件像积木一样组合即可
3、不断迭代表单控件和视图控件以满足不断增加的需求
4、一个管理平台80%都可以积木组合完成，20%的功能还得diy完成，系统支持一键生成表单页面自己修改
5、集成钉钉工作流
5、集成BI数据分析
6、可以生成安装包，让小白用户快速本地安装部署
7、Oracle版本支持
8、积木特色Dashboard，只要拖拽以及懂sql即可快速定制看板

第一步：设计一张表(需要一定复杂度表)，比如这张：demo2_test，约20个字段(包含:省市区、编辑字段，转义字段，关联字段，文件图片上传等)

第二步：使用积木 平台维护=》元数据管理=》导入元数据 选择刚设计的表 输入:元对象名称、元对象编码，点击‘保存’

第三步：再此列表查看此对象相关信息，下部设计元对象的字段信息，比如某些字段控件类型需要设置成文件框，某些字段需要需要设置编辑框等(当然不设置也可以只是出现的基本是输入框)

第四步：元对象字段设置完成，设置菜单，主要是设置视图模式，平台维护=》菜单管理=》新增功能 业务模板选择：单表，元对象选择：刚才创建的元对象
